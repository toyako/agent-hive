## 独有能力

1. **多 Runtime 协作** — 不同 AI 执行不同角色
2. **独立 Review** — 执行者 ≠ 审查者
3. **Revision Loop** — 审查不通过自动重试
4. **Runtime Benchmark** — 量化各 Runtime 能力
5. **Context Compression** — 防止上下文爆炸

## 行业已有能力

1. Memory System — Hermes 已有
2. Observability — OpenHands 已有
3. CLI 交互 — Claude Code/OpenCode 已有
4. Setup Wizard — 大多已有

## 用户价值主张

### 对于个人开发者
"用一个命令，让多个 AI 帮你写代码、审查代码。"

### 对于团队
"标准化 AI 协作流程，可观察、可回溯。"

### 对于企业
"不绑定单一 AI 供应商，Runtime 可替换。"

## Why Not X?

### Why not Claude Code?
Claude Code 只用 Claude。我们用 Claude + Codex + Hermes + 任何 API。

### Why not OpenCode?
OpenCode 只用一个 Runtime。我们支持多 Runtime 协作。

### Why not Hermes?
Hermes 是单 Runtime agent。我们是多 Runtime 编排。

### Why not OpenHands?
OpenHands 侧重执行。我们侧重协作 + 审查 + 修订。

### Why not tctinh/agent-hive?
tctinh 的是 OpenCode 插件。我们是独立框架。

## 一句话总结

**RuntimeOS: 让多个 AI 像团队一样协作。**
